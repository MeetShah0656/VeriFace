VeriFace Architecture Breakdown

A guide explaining the tech stack, directories, and real-time biometric pipeline powering VeriFace.

⚙️ Hybrid Serverless Edge System

VeriFace utilizes **Edge Biometrics**. All camera ingestion, face bounding estimation, blur filtering, crop alignment, and convolutional feature embedding happen completely in the user's browser via high-speed WebAssembly compiles.

Firestore rules and Firebase Auth manage access on the cloud database, enabling secure, multi-device backup syncs whenever network signals are present.

⚡ Biometric Recognition Pipeline
01 Camera Ingestion

Webcam stream is routed directly into an HTML5 <video> viewport.

02 MediaPipe BlazeFace

Evaluates camera frames every 250ms via WebAssembly to estimate 2D coordinates of facial features.

03 Biometric Quality Filter

Applies Laplacian variance to evaluate sharpness (blur) and relative luminance checks to filter bad lighting.

04 Padded Square Crop

Identifies bounding center, calculates maximum face dimensions, adds 15% margin, and draws a clean `112x112` canvas crop.

05 ArcFace ONNX Inference

Normalizes pixels to `[-1, 1]` and passes them to ArcFace CNN using ONNX Runtime Web to generate a 512-dimension vector.

06 Cosine Similarity Match

Calculates dot-product vectors against stored embeddings in IndexedDB. Any match score > 0.65 resolves the user.

🛠️ Technology Stack Detailed Breakdown

Category Technology / Library Purpose & Under-the-Hood Mechanics
Core Framework Next.js 15 (App Router) Base React framework. Handles asset layouts, client-side routing transitions, Turbopack compiling, static pre-rendering, and page assets load.
UI Engine React 19 Orchestrates interactive states, handles rendering updates, and ties canvas/video hooks together during runtime loops.
Type Safety TypeScript Enforces typed models for student datasets, rosters, sessions, and biometric similarity parameters.
Face Detection MediaPipe Tasks Vision Loads the `blaze_face_short_range.tflite` model in the browser via WebAssembly to track faces and estimate 6 key facial coordinates.
ONNX Execution ONNX Runtime Web Injects multi-threaded WASM with SIMD instructions to perform heavy convolutional neural network operations on the browser CPU.
Face Model ArcFace CNN (`arc.onnx`) A feature extraction neural network model. Translates canvas crop shapes into a 512-dimensional floating vector representing face metrics.
Offline Cache IndexedDB (`idb` wrapper) Browser-level database storing student metadata, rosters, sync status indices, settings, and local face embeddings.
Cloud DB Cloud Firestore Remote NoSQL server database managing secure persistent backups. Syncs records when the client transitions from offline to online.
Authentication Firebase Auth Handles secure login profiles for teachers (supporting Google Sign-In and standard Email).
Styling Tailwind CSS v4 Utility stylesheet compiler driving custom responsive grid displays and modern dark modes.
Animations Framer Motion Fires interactive screen transitions, loading indicators, and card modal actions.
Charts Recharts Renders real-time visual charts displaying attendance rates and class historical figures.
Data Export SheetJS (`xlsx`) Compiles attendance grids client-side and formats them into download-ready Excel files.

📁 Project Directory Map

Directory Path Primary Purpose Key Contents & Scope
app/ Next.js Page Routes & UI Framework Contains page routes: `/login`, `/dashboard`, `/classes`, and `/students/register`. Also holds root layout page bindings.
components/ Shared Global UI components Custom elements such as layout sidebar panels, headers, search bars, and dialog modals.
contexts/ Application Global States Manages teacher sign-in session contexts (`auth-context.tsx`) and client-side server query caches.
hooks/ Reusable React Life Hooks `useFaceRecognition.ts` (manages camera feeds, quality verification, and crop pipelines) and online status trackers.
public/ Static Web Resources Hosts local WebAssembly bundles for MediaPipe Face Detector and ONNX WASM runtimes.
services/ Engines & Business Logic APIs IndexedDB schema actions (`db.ts`), ArcFace models loading and cosine checks (`arcfaceService.ts`), and sync monitors (`syncService.ts`).
utils/ Math & Quality Calculations Filters poor inputs using Laplacian checks (blur) and relative brightness metrics (`faceQuality.ts`).
